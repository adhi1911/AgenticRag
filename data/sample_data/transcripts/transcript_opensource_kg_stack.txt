For Layer 1, data integration, we have several choices:

Apache NiFi is a powerful data ingestion tool. It's visual, which helps non-engineers understand data pipelines.

A basic flow might be:
```
GetFile → ParseJSON → ConvertRecord → PutDatabase
```

Advantages:
- Visual flow design
- Handles 100s of concurrent flows
- Built-in error handling
- Data lineage tracking

Disadvantages:
- Steep learning curve for complex scenarios
- Resource intensive
- Operational overhead

Alternative: Apache Kafka for real-time data streaming.

Kafka topics might be:
- products (product updates from ERP)
- suppliers (supplier data from sourcing system)
- reviews (customer reviews from multiple channels)

Producers push data to topics. Processors consume and transform. Consumers push to storage.

The advantage: Decoupled systems, scalability, replay capability.

Entity linking is critical and hard. Two approaches:

Approach 1: String matching with candidates
```python
from fuzzywuzzy import fuzz

entities = {"Apple Inc", "iPhone 15 Pro", "Samsung Galaxy S24"}
query = "Appel Inc"

matches = [(e, fuzz.ratio(query, e)) for e in entities]
matches.sort(key=lambda x: x[1], reverse=True)
print(matches[0])  # ("Apple Inc", 96)
```

Approach 2: Semantic similarity with embeddings
```python
from sentence_transformers import SentenceTransformer, util

model = SentenceTransformer('all-MiniLM-L6-v2')

entities = ["Apple Inc", "Apple Inc.", "Apple Computer Company"]
query_embedding = model.encode("Apple Inc", convert_to_tensor=True)

for entity in entities:
    entity_embedding = model.encode(entity, convert_to_tensor=True)
    similarity = util.pytorch_cos_sim(query_embedding, entity_embedding)
    print(f"{entity}: {similarity.item():.3f}")
```

Output on a typical run:
- "Apple Inc": 1.000
- "Apple Inc.": 0.998
- "Apple Computer Company": 0.945

Good, but not perfect for variant names.

Hybrid approach:
```python
def entity_linker(query, candidates, threshold=0.85):
    # Try exact match
    if query in candidates:
        return query
    
    # Try fuzzy match
    fuzzy_matches = [(c, fuzz.ratio(query, c)/100) 
                     for c in candidates]
    if fuzzy_matches[0][1] > 0.95:
        return fuzzy_matches[0][0]
    
    # Try semantic
    embeddings = model.encode([query] + candidates)
    similarities = [
        util.pytorch_cos_sim(embeddings[0], embeddings[i+1])
        for i in range(len(candidates))
    ]
    best_idx = similarities.index(max(similarities))
    
    if similarities[best_idx] > threshold:
        return candidates[best_idx]
    
    return None  # No match found
```

Creating a knowledge graph:
```python
from neo4j import GraphDatabase

driver = GraphDatabase.driver("bolt://localhost:7687",
    auth=("neo4j", "password"))

def create_product_kg(tx, products, suppliers, relations):
    # Create product nodes
    for product in products:
        tx.run("""
            CREATE (p:Product {
                id: $id,
                name: $name,
                price: $price,
                category: $category
            })
        """, id=product['id'], name=product['name'],
            price=product['price'], category=product['category'])
    
    # Create supplier nodes
    for supplier in suppliers:
        tx.run("""
            CREATE (s:Supplier {
                id: $id,
                name: $name,
                country: $country
            })
        """, id=supplier['id'], name=supplier['name'],
            country=supplier['country'])
    
    # Create relationships
    for relation in relations:
        tx.run("""
            MATCH (s:Supplier {id: $supplier_id})
            MATCH (p:Product {id: $product_id})
            CREATE (s)-[r:SUPPLIES {since: $since}]->(p)
        """, supplier_id=relation['supplier'],
            product_id=relation['product'],
            since=relation['since'])

with driver.session() as session:
    session.execute_write(create_product_kg,
        products, suppliers, relations)
```

For simpler cases, PostgreSQL with JSON columns works fine:
```sql
CREATE TABLE entities (
    id SERIAL PRIMARY KEY,
    type VARCHAR(50),
    properties JSONB
);

INSERT INTO entities (type, properties) VALUES
('Product', '{"name": "iPhone 15", "price": 999}'),
('Company', '{"name": "Apple Inc", "country": "USA"}');

CREATE TABLE relationships (
    source_id INT,
    target_id INT,
    rel_type VARCHAR(50),
    properties JSONB
);

SELECT e1.properties->>'name', r.rel_type, e2.properties->>'name'
FROM relationships r
JOIN entities e1 ON r.source_id = e1.id
JOIN entities e2 ON r.target_id = e2.id;
```

Comparison:
- Neo4j: Best for heavy graph traversal, complex queries
- RDF stores: Best for semantic reasoning, linked data
- PostgreSQL: Best for simplicity, existing infrastructure

For our scenario, I'd choose Neo4j for its query performance on product relationships.

Layer 4: Query & Retrieval

Neo4j Cypher query language:
```cypher
// Find products supplied by Apple
MATCH (s:Supplier {name: "Apple Inc"})
       -[:SUPPLIES]->(p:Product)
RETURN p.name, p.category ORDER BY p.price DESC

// Find alternatives (products with same category)
MATCH (p1:Product {name: "iPhone 15"})-[:IN_CATEGORY]->(c:Category)
      <-[:IN_CATEGORY]-(p2:Product)
WHERE p1 <> p2
RETURN p2.name, p2.price

// Graph traversal (find suppliers of suppliers)
MATCH (s1:Supplier)-[:SUPPLIES]->(p:Product)
      <-[:MANUFACTURED_BY]-(material:Material)
      -[:SUPPLIED_BY]->(s2:Supplier)
RETURN DISTINCT s2.name
```

import streamlit as st
from neo4j import GraphDatabase

driver = GraphDatabase.driver(...)

st.title("Product Knowledge Graph")

search_term = st.text_input("Search for a product")

if search_term:
    with driver.session() as session:
        results = session.run("""
            MATCH (p:Product)

            RETURN p.name, p.category, p.price
            LIMIT 20
        """, pattern=f"(?i).*{search_term}.*")
        
        df = pd.DataFrame([dict(r) for r in results])
        st.dataframe(df)

        selected = st.selectbox("Choose a product", df['name'].tolist())
        if selected:
            suppliers = session.run("""
                MATCH (s:Supplier)-[:SUPPLIES]->(p:Product)
                WHERE p.name = $name
                RETURN s.name, s.country
            """, name=selected)
            
            st.subheader("Suppliers")
            st.table(suppliers)
